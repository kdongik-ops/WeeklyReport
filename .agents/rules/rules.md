# 주간보고 업무 워크플로우

주간보고 작성 시 다음 프로세스를 준수합니다.

## 공통 원칙
- **읽기 전용 명령 자동 실행**: `npm list`, `dir`, `type` 등 시스템 상태를 읽기만 하는 명령은 사용자 승인 없이 `SafeToAutoRun: true`로 설정하여 자동으로 진행합니다.

---

## 1단계: 초안 생성
1. **소스**: `raw_data` 폴더에서 해당 주차 텍스트 읽기.
2. **초안 생성**: `report_rules.md`의 요약/그룹화 원칙을 적용하여 마크다운(.md) 파일 생성. (주의: 사용자가 축약된 내용의 출처를 파악할 수 있도록, 초안 작성 시 **각 항목별로 원본 데이터(RAW)와 요약된 결과(요약)를 함께 비교할 수 있는 형식**으로 보여주어야 함)
3. **파일명**: `output/[주차]_초안.md`
4. **검토 요청**: 사용자에게 내용 검토(오타, 누락 등) 요청.

## 2단계: 요약 검증 (RAW ↔ 요약 비교 분석)
1. **트리거**: 1단계 초안 생성 완료 후, 사용자 검토 요청 전에 자동 수행.
2. **검증 수행**: 초안의 각 항목에 대해 `[RAW]` 원본과 `[요약]` 결과를 1:1 비교 분석하여 다음 항목을 점검한다.
   - **누락 확인**: RAW 데이터에 존재하나 요약에서 빠진 내용이 없는지 확인.
   - **과장 표현 확인**: RAW 데이터의 원래 표현보다 과장되거나 확대 해석된 부분이 없는지 확인.
   - **추측 표현 확인**: RAW 데이터에 명시되지 않은 내용을 추측하여 요약에 추가한 부분이 없는지 확인 (예: RAW에 '완료' 표현이 없는데 요약에 '완료'로 기재한 경우).
3. **결과 제시**: 검증 결과를 **표(테이블) 형식**으로 정리하여 사용자에게 제시한다. (항목별: RAW 원문 | 요약 내용 | 검증 결과(✅ 적합 / ⚠️ 수정 필요) | 비고)
4. **수정 반영**: 검증에서 문제가 발견된 항목은 초안에 자동 수정 반영 후, 수정 사항을 사용자에게 안내한다.

## 3단계: 최종본 생성
1. **트리거**: 사용자의 "이상 없음" 또는 "최종본 생성" 요청 시.
2. **생성**: 초안에 `report_rules.md`의 위키 디자인 규칙(HTML, 볼드 등) 적용. 단, 최종본 생성 시에는 초안의 **`[RAW]` 내용은 제외하고 `[요약]` 부분만을 취합**하여 생성한다.
3. **파일명**: `output/[주차].html`

---

## 마무리
1. `output/[주차].html` 파일 브라우저로 열기.
2. 텍스트 복사(`Ctrl+A`, `Ctrl+C`).
3. 사내 위키에 붙여넣기(`Ctrl+V`).
